Introducing Merkur: Our Prototype Smart Home AI

Published: Sep 12, 2025 | Category: AI

Smart homes are no longer a futuristic dream—they are here. Yet, controlling every device intelligently and naturally has remained a challenge. This is where Merkur comes in: a prototype home assistant that combines voice control, OpenHAB integration, and cutting-edge AI to manage your entire smart home.

The Prototype Setup

Merkur currently runs on a standard PC, although its architecture is lightweight enough to eventually run on a Raspberry Pi or similar small devices. For hardware, it uses a microphone for capturing voice commands and a speaker for responses. The focus so far has been on building a fully functional proof-of-concept that demonstrates the system’s capabilities with real devices.

OpenHAB Integration

OpenHAB serves as the backbone of Merkur’s device control. Communication is handled via REST API, making it compatible with virtually any device supported by OpenHAB. In our prototype, we’ve successfully integrated:

  • A smart lamp for lighting control
  • Climate devices such as thermostats and air conditioners
  • Xiaomi smart home products, including sensors and smart plugs

Since Merkur talks directly to OpenHAB, it can theoretically control any device in your smart home ecosystem, from lights to security systems, without writing custom code for each new product.

AI-Powered Voice Control

The heart of Merkur’s intelligence is an advanced AI engine. While the exact integration method is currently proprietary, the AI allows Merkur to:

  • Understand natural language commands, including complex multi-step instructions
  • Maintain context across multiple commands, enabling conversational control
  • Execute orchestrated actions across multiple devices simultaneously

For example, a single command like “Dim the living room lights, set the temperature to 22°C, and start the evening playlist” is fully understood and executed by Merkur without needing multiple steps or manual configuration.

Features in Action

Even as a prototype, Merkur demonstrates a wide array of capabilities:

  • Voice-first control: Speak naturally, and Merkur will understand and act accordingly.
  • Full OpenHAB compatibility: Any device already integrated in OpenHAB can be controlled without extra work.
  • Context awareness: Merkur can handle follow-up commands, remembering what you just asked it to do.
  • Custom automations: Prototype users can define multi-device routines, like “Good night” to turn off lights, close blinds, and adjust the thermostat.

Building Merkur

Developing Merkur has been both exciting and challenging. Key steps included:

  1. Setting up REST-based communication with OpenHAB to allow seamless device control.
  2. Integrating AI to parse natural language into actionable commands while maintaining context.
  3. Creating a lightweight, modular architecture so the system can eventually run on smaller hardware like a Raspberry Pi.
  4. Ensuring the prototype is flexible enough to scale to new devices and automations without rewriting code.

While Merkur is already fully functional, the team is continuing to refine the AI integration and polish the user experience. In the coming months, we expect to finalize the system and release it as a consumer-ready smart home assistant.

Looking Ahead

Merkur represents the future of intelligent home automation: a system that is voice-first, AI-driven, and compatible with any OpenHAB-supported device. Our prototype shows that controlling your smart home can be as natural as talking to a friend, and we’re just getting started.
